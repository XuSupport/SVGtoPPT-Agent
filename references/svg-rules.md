# SVG Rules

## 目标

生成能直接预览、适合导入 PPT 的单页 SVG。

## 画布

- 默认使用 16:9
- 推荐：
  - `width="1600" height="900"`
  - `viewBox="0 0 1600 900"`
- 所有页保持同一画布尺寸

## 结构

- 每页一个独立 `<svg>`
- 使用 `<g>` 组织区块
- 使用 `<rect>`、`<circle>`、`<line>`、`<path>`、`<text>` 等基础元素
- 不要使用脚本
- 不要依赖外部 CSS

## 文本

- 文本必须直接写在 SVG 中
- 不要把文字转成路径，除非用户明确要求
- 标题、正文、注释至少有三档字号层级
- 保证投影可读性，避免过小字号
- 文本来源必须是精修后的上屏文案，而不是原始长句

### 文本预算

- 页面主标题：18-28 个汉字或等效信息量
- 副标题：24-40 个汉字或等效信息量
- 核心结论大字区：建议 2-4 行，每行 10-16 个汉字
- 常规 bullet：建议 16-28 个汉字
- 卡片标题：建议 8-14 个汉字
- 卡片正文：建议 2 行内，每行 14-22 个汉字
- 页脚来源：优先 1 行来源 + 1 行备注，不写成长段

如果任一容器放不下文本，先改写文本，不要先缩字号。

## 样式

- 使用内联 `fill`、`stroke`、`font-size`、`font-weight`
- 控制阴影、渐变和滤镜的使用
- 能不用复杂滤镜就不用
- 颜色数量受控，优先统一风格
- 默认只保留 1 个主视觉焦点和 1 个次级强调区
- 高饱和强调色块不要同时出现在过多区域
- 背景大字、噪声纹理、网格、流光线条都属于可删项，不属于必需项
- 整套 deck 应统一配色逻辑，但不同页面允许使用不同卡片语法
- 不要让每页都复用同一种实心卡片加同一种标题条
- 先定义风格令牌，再绘制页面；不要一边画一边临时发明风格
- 若当前页属于某个页面类型，应优先遵守该页型覆盖规则，再做局部发挥

### 渐变规则

- 渐变优先用于关键结论、终点状态、总结金句和少量标签
- 一套 deck 建议只定义 2-3 组主渐变
- 渐变要服从整体色彩系统，不要随机拼接高饱和颜色
- 若渐变会降低文字对比度，优先保留纯色底
- 光晕、渐变描边和弱背景辉光只能做气氛层，不能成为主信息
- 一页里强渐变区域默认不超过 2 处

## 布局

- 主信息块优先位于视觉重心
- 容器之间保留明确间距
- 不要让线条、标签和正文互相遮挡
- 若内容太多，优先删减或拆页
- 不要为了塞下文本而把字号压到读不动
- 先保留层级与留白，再决定是否裁撤低优先级信息
- 如果装饰元素和正文争抢视线，默认删装饰
- 大字背景只有在明显弱于主体时才允许存在
- 底部流程、页脚和角标必须保留安全边距，不能压边堆叠
- 相邻页面的卡片节奏不应完全重复
- 开场页、流程页、指标页、总结页应有不同的模块组织方式
- 连续三页同一种页面语法默认视为失败
- 若 deck 中出现大量同构页面，应优先调整页型分配，而不是继续往原模板上加小变化

## 图示

- 流程图：优先用规则箭头和容器
- 对比图：优先双栏或分组卡片
- 指标页：优先数值卡 + 简短说明
- 结构页：优先容器分层和连接关系
- 如果页面内容显得单调，优先增加手作结构图，而不是先找外部图片
- 若 skill 模板库已有合适骨架，优先复用模板结构

## 预览图复检

- SVG 完成后必须渲染成 PNG 或缩略图，再做视觉检查
- 不要因为代码结构整齐就假设画面也整齐
- 如果渲染图第一眼感觉乱，说明层级、装饰或节奏有问题
- 判断标准以渲染图为准，不以作者主观预期为准
- 整套页完成后，还要做一次 deck 级别的横向复检
- 如果 10 页看起来像同一页的不同文案版本，说明卡片语法仍然不够分化

## 禁止项

- 禁止输出多页混在一个 SVG
- 禁止依赖网页脚本或动画
- 禁止使用外链图片作为关键内容
- 禁止用过度装饰代替信息组织
- 禁止把文章式长段落直接塞进卡片、注释框或流程节点
- 禁止在用户未确认风格前直接批量生成整套页面
- 禁止跳过渲染图复检直接宣称页面完成
