# Visual Review

## 目标

把 SVG 渲染成图片后，从观众视角检查页面是否和谐、稳定、清楚。

默认使用 `scripts/render_svg_preview.sh` 通过 `sharp` 生成 PNG 预览图，避免依赖 `qlmanage` 这类平台特定缩略图行为。

## 必查项

- 主体是否一眼可见，还是多个区域都在抢第一眼
- 标题、核心结论、主图、流程和右侧信息块是否互相打架
- 背景大字、网格、纹理、发光和渐变是否压过正文
- 页面是否存在“没有重叠，但看起来仍然很乱”的情况
- 卡片之间的距离是否足够，底部流程是否过于贴边
- 标签、角标和风险章是否数量太多或颜色太跳
- 渐变是否只用在关键位置，还是已经铺得到处都是
- 页面有没有因为“想做精致”而堆出过多小巧思
- 结构图、箭头、矩阵点位和文字之间是否有遮挡或贴得过近
- 这一页在整套中的页型身份是否清楚，还是和前后页几乎同构

## 评分参考

单页可按以下维度粗评分：

- Layout Balance: 25%
- Readability: 25%
- Typography: 20%
- Information Density: 20%
- Color Harmony: 10%

通过门槛建议：

- 总分 `>= 7.0`
- Layout Balance `>= 6`
- Readability `>= 6`
- 无关键重叠或 critical issue

修正语义：

- `>= 7.0`：通过
- `5.0 - 6.9`：允许 1-2 轮定向修复
- `3.0 - 4.9`：最多 1 轮，优先换结构
- `< 3.0`：直接重做，不要补丁式微调

## 常见失败模式

- 用背景大字制造气势，但背景字比正文更抢眼
- 同时出现太多高饱和色块，导致没有真正主次
- 主标题、角标、右上风险标签和底部流程都在争夺注意力
- 网格、发光、渐变、分隔线全部一起上，画面噪音过高
- 版面元素都“能放下”，但节奏失衡，看着拥堵
- 所有页面都长得像同一张卡片模板，缺少页面身份
- 为了显高级到处加渐变，结果重点被平均掉

## 修正顺序

1. 先删最抢戏的装饰
2. 再降低背景和次级元素存在感
3. 再减少高饱和强调区数量
4. 再拉开主要区块之间的距离
5. 最后才调局部字号、线条和圆角
6. 整套看仍然同质化时，回到页面功能层重新分配卡片语法
7. 若问题来自 deck 节奏失衡，回到 `page_family_plan.md` 重新分配，而不是只改颜色

## 完成标准

- 第一眼能明确知道先看哪里
- 第二眼能顺畅找到支撑信息
- 没有任何背景元素比正文更强
- 页面虽然有设计感，但不会让人觉得乱
- 整套页放在一起时，能看出统一风格，也能看出页面之间的性格差异
- 所有结构图和文本之间没有重叠、压线或安全边距不足的问题
